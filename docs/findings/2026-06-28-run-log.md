- **KEY FINDING — the local tier is DUAL-ENGINE.** GLM-4.7-Flash-AWQ (cyankiwi compressed-tensors)
  **cannot load on SGLang** (open bug #25331: GLM's MLA init allowlists only awq/awq_marlin/moe_wna16,
  not compressed-tensors → `ReplicatedLinear has no attribute 'weight'`; no fix in any release/dev/nightly,
  PR closed unmerged). A newer SGLang does NOT solve it. Root-cause fix = **vLLM**, where
  compressed-tensors W4A16 is the native Marlin-INT4 path on sm_120. Implication for anvil-serving:
  deploy/multiplexer must support BOTH SGLang and vLLM backends (the T009 multiplexer's injectable
  backend seam already allows this). NOT a shortcut/fallback — this is the correct engine for the build.
- **GLM thinking-default gotcha:** empty `content` + `finish:length` under small max_tokens; disable via
  `chat_template_kwargs {"enable_thinking": false}` (or parse `reasoning_content`). Confirmed.
- **Multiplexer now multi-engine** (PR #7, `a2e73fd`): registers all 3 (qwen3-coder-local sglang/gpu1,
  glm-4.7-flash vllm/gpu0, qwen3-14b-fast sglang/gpu0); per-engine docker dispatch (vLLM entrypoint
  override; host_path mounts; CUDA_VISIBLE_DEVICES=UUID isolation for Docker-Desktop WSL2). Both fast
  options serve on demand, single-resident-swapped on GPU 0. 2 purify rounds caught the doubled-vLLM
  entrypoint, the host==container mount, and the `--gpus device=N` WSL2 no-op. Sentinel 7/7.

### ⚠️ anvil finding (bookkeeping bug)
`anvil prd parse` does NOT refresh `verification`/`required_evidence` on **already-created** tasks —
only on first creation. After the task-graph repair (PRD verification fixed to `anvil_serving.*`),
a re-parse reported "Revised 15 tasks" but left every existing task's `required_evidence` pointing at
the original fictional commands (`python -m src.serving.multiplexer …`). So `anvil submit` flags the
evidence-gate INCOMPLETE against commands that can never pass, while the real verification (PR +
Sentinel) is green. Tasks with claims/evidence (T003/T004/T009) can't be deleted+recreated (FK-protected),
so the stale gate is stuck for them. **Real verification gate for this loop = adversarial purify +
Sentinel + PR merge, NOT the anvil evidence-gate.** Worth an anvil fix: have `prd parse` update
verification on existing tasks, or add a `task update-verification` command.
